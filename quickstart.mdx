This guide takes you from zero to a working Killa store in minutes. You'll create a store, read and update state, subscribe to changes, and wire it up to a React component.

## Prerequisites

Install Killa before starting:

```bash
npm install killa
```

For React usage, also ensure React 18+ is installed:

```bash
npm install react react-dom
```

## Vanilla quickstart

<Steps>
  <Step title="Create your first store">
    Pass a plain object to `createStore`. The object becomes your initial state.

    ```js
    import { createStore } from 'killa'

    const store = createStore({ counter: 0 })
    ```

    <Note>
      The state inside a Killa store is **immutable** — you can only change it through `setState`. The `store` object itself is frozen in production.
    </Note>
  </Step>

  <Step title="Read state">
    Call `getState()` to get the current state. It returns a deep clone — safe to read, but mutations won't affect the store.

    ```js
    store.getState()
    // { counter: 0 }
    ```
  </Step>

  <Step title="Update state">
    Call `setState` with a function that receives the current state and returns the new (partial) state. Killa merges the result with the existing state.

    ```js
    store.setState((state) => ({
      ...state,
      counter: state.counter + 1
    }))

    store.getState()
    // { counter: 1 }
    ```

    <Tip>
      You can return only the fields you want to change — Killa performs a shallow merge automatically.
    </Tip>
  </Step>

  <Step title="Subscribe to changes">
    Use `subscribe` to react to state changes. Pass an optional selector to narrow which changes trigger the callback.

    ```js
    // Global subscriber — fires on any state change
    store.subscribe((state, prevState) => {
      console.log('counter changed:', prevState.counter, '→', state.counter)
    })

    // Selective subscriber — fires only when counter changes
    store.subscribe((state, prevState) => {
      console.log('counter:', state.counter)
    }, (state) => state.counter)

    store.setState((state) => ({ ...state, counter: state.counter + 1 }))
    // Logs: counter changed: 1 → 2
    // Logs: counter: 2
    ```

    Subscribe returns an unsubscribe function:

    ```js
    const unsub = store.subscribe((state) => console.log(state))
    unsub() // removes the subscriber
    ```
  </Step>

  <Step title="Reset or destroy">
    Reset the store to its initial state, or clear all subscribers:

    ```js
    store.resetState()            // back to { counter: 0 }
    store.resetState({ notes: [] }) // overwrite with new state

    store.destroy()               // clear all subscribers
    ```
  </Step>
</Steps>

## React quickstart

<Steps>
  <Step title="Create a shared store">
    Define your store outside of any component so it can be shared across the entire app.

    ```js
    import { createStore } from 'killa'

    export const counterStore = createStore({ counter: 0, filter: '' })
    ```
  </Step>

  <Step title="Connect to a component with useStore">
    Import `useStore` from `killa/react` and pass in your store and a selector. The selector determines which part of the state this component subscribes to.

    ```jsx
    import { useStore } from 'killa/react'
    import { counterStore } from './store'

    export function Counter() {
      // Only re-renders when `counter` or `filter` changes
      const [state, setState] = useStore(counterStore, (state) => ({
        counter: state.counter,
        filter: state.filter
      }))

      const handleIncrement = () => {
        setState((state) => ({
          ...state,
          counter: state.counter + 1
        }))
      }

      return (
        <div>
          <p>Counter: {state.counter}</p>
          <button onClick={handleIncrement}>+1</button>
        </div>
      )
    }
    ```

    <Note>
      `useStore` returns a `[state, setState]` tuple. The `setState` function is the same as `store.setState` — it receives a function and merges the returned value.
    </Note>
  </Step>

  <Step title="Add a second component reading a different slice">
    Selective subscriptions mean components only re-render for the state they care about:

    ```jsx
    export function FilterInput() {
      // Only re-renders when `filter` changes — not when `counter` changes
      const [state, setState] = useStore(counterStore, (state) => state.filter)

      return (
        <input
          value={state}
          onChange={(e) => setState((s) => ({ ...s, filter: e.target.value }))}
          placeholder="Filter..."
        />
      )
    }
    ```
  </Step>
</Steps>

## Using internal actions

Instead of calling `setState` from outside the store, you can define actions inside the initializer:

```js
import { createStore } from 'killa'

const store = createStore((get, set) => ({
  count: 0,
  inc: () => set(() => ({ count: get().count + 1 })),
  dec: () => set(() => ({ count: get().count - 1 })),
  reset: () => set(() => ({ count: 0 }))
}))

store.getState().inc()  // count: 1
store.getState().inc()  // count: 2
store.getState().dec()  // count: 1
store.getState().reset() // count: 0
```

In React:

```jsx
const [{ count, inc, dec }, setState] = useStore(store, (s) => s)

return (
  <div>
    <button onClick={dec}>-</button>
    <span>{count}</span>
    <button onClick={inc}>+</button>
  </div>
)
```

## TypeScript

Killa is TypeScript-first. Pass your state type as a generic:

```ts
import { createStore } from 'killa'

interface AppState {
  counter: number
  filter: string
  items: string[]
}

const store = createStore<AppState>({
  counter: 0,
  filter: '',
  items: []
})

// Fully typed — TypeScript knows the shape of state
store.setState((state) => ({
  counter: state.counter + 1 // state.counter: number ✓
}))
```
